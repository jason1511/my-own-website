Bike Store Inventory & Sales App

Case Study · C#, .NET WinForms, SQLite

A desktop application built to support day-to-day operations for a small electric bike retail business: inventory tracking, transaction logging, and basic reporting. Designed to be reliable and easy to use.

Features Architecture Learnings

The Problem

Small businesses often track inventory and sales across spreadsheets or notebooks, which can lead to inconsistent records, duplicated work, and low visibility of stock movement.

Challenges

  • Manual tracking makes it easy to lose or duplicate records
  • Hard to quickly check stock availability
  • Sales and service history is scattered
  • Simple reporting takes time

Goal

Build a simple desktop app that supports day-to-day operations with structured data storage and an easy UI, while staying lightweight and reliable.

Key Features

Built around the core workflows: manage stock, record transactions, and review basic summaries.

Inventory Management

  • Create and update product records
  • Track quantities and item details
  • Designed for quick lookup and accuracy

Sales & Transactions

  • Record sales transactions consistently
  • Maintain a transaction log for review
  • Supports structured history of changes

Reporting (Basic)

  • Simple summaries to support decision-making
  • Designed to be clear and easy for staff

Usability & Reliability

  • Focused UI for non-technical users
  • Offline-first with local SQLite database
  • Iterated based on user feedback

Architecture & Implementation

The app is built with a straightforward structure to keep it maintainable and easy to extend.

Core Design Choices

  • WinForms UI for fast desktop workflows
  • SQLite for local, reliable storage
  • Clear separation between UI and data logic (repository-style patterns)
  • Incremental updates based on real usage

What I Would Add Next

  • More advanced reports (export / summaries)
  • Better search and filtering
  • Role-based access
  • Optional cloud sync

Challenges & Learnings

This project strengthened my ability to translate user needs into software features and iterate reliably.

What I Learned

  • Gathering requirements and refining scope
  • Designing practical UI flows for non-technical users
  • Working with structured data using SQLite
  • Iterating improvements based on feedback

What I’m Proud Of

  • End-to-end ownership: plan → build → test → refine
  • A real-world use case with practical value
  • Clean, maintainable structure for future improvements
